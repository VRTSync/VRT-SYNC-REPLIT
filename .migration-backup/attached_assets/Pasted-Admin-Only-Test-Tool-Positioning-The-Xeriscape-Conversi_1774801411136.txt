Admin-Only Test Tool Positioning

The Xeriscape Conversion Planner should initially live only in the admin web portal and be available only to VRTSync internal/admin users. Its purpose in this phase is to test the workflow, prove the value of the calculations, and refine how results are presented before exposing it to HOA or property manager users.

In this first version, admins would be able to:

view mapped bluegrass KML polygons

select one or multiple polygons

group selected polygons into planning phases or estimate groups

calculate total square footage

apply a default or editable conversion cost per square foot

apply a default or editable annual water savings assumption

generate estimated conversion cost, annual savings, and payback period

save or export rough planning scenarios for internal use or proposal building

Why This Is The Right First Step

This keeps the feature safely in test mode while still making it highly useful. It gives your team a way to use it in real sales conversations, proposal creation, and internal planning without needing to fully productize the customer experience yet.

It also helps you answer important questions first:

Do the calculations feel credible?

Does grouping work the way real estimates need it to?

What outputs are most helpful in sales conversations?

What assumptions need to stay editable?

What do boards and PMs actually respond to?

Overview Scope Paragraph

Build a new admin-only Xeriscape Conversion Planner inside the VRTSync Admin Web Portal as an internal test tool for evaluating turf-to-xeriscape conversion opportunities using mapped KML bluegrass polygons. This tool should allow admin users to view imported bluegrass polygon geometry on the map, select individual or multiple polygons, organize selections into named planning groups or phases, automatically calculate square footage from polygon geometry, apply configurable planning assumptions such as conversion cost per square foot and annual water savings per square foot, and generate rough planning outputs including estimated project cost, estimated annual water savings, and projected payback period. Slice 1 — Admin-Only Xeriscape Conversion Planner Foundation
What & Why

Build the foundational admin-only test version of the Xeriscape Conversion Planner in the VRTSync Admin Web Portal. The goal of this first slice is to create the basic structure needed to display bluegrass polygons, select them, calculate square footage, and generate simple planning estimates using editable assumptions. This slice should prove the core concept works before adding saved scenarios, exports, or more advanced comparisons.

Done Looks Like

A new admin-only page exists in the Admin Web Portal called Xeriscape Planner

Only Super Admin / Admin portal users can access this page

The page contains:

a map panel showing test bluegrass polygons loaded from KML-derived geometry or a temporary converted GeoJSON source

a side planning panel showing assumptions and calculation outputs

a polygon list or selection summary tied to the map

Admin users can:

view all loaded bluegrass polygons on the map

click a polygon to select it

select multiple polygons

see selected polygons visually highlighted on the map

see a running total of selected square footage

apply a default conversion cost per SF

apply a default annual water savings per SF

see calculated outputs for the current selection:

total selected area

estimated conversion cost

estimated annual water savings

estimated water-only payback period

This first slice does not yet need:

saved groups/phases

scenario comparison tables

PDF export

customer-facing access

persistent saved estimates across sessions

integration with live customer data

AMI or water utility integrations

Functional Requirements
1. Admin Portal Placement + Access Control

Add a new admin-only route, something like /web/admin/xeriscape-planner

Route must only be accessible from the admin shell

Do not expose this page in HOA, PM, or contractor portals

Mark internally as a test/planning tool

2. Map Display

Render imported bluegrass polygons on the map

Support polygon highlighting for hover and selected state

Clicking a polygon toggles selection on/off

Multiple polygons can be selected at once

Map should zoom to the extent of loaded polygons on initial load if possible

3. Polygon Data Model

Each polygon should have at minimum:

internal id

name or fallback label

geometry

computed square footage

optional source metadata if available from KML

If the KML does not provide a usable name field, generate a fallback like:

Bluegrass Area 1

Bluegrass Area 2

4. Area Calculation

Compute square footage from the polygon geometry, not from a manual field

Display square footage per selected polygon and total square footage for all selected polygons

Round displayed values cleanly for user readability, but keep internal precision consistent

5. Assumptions Panel

Include editable admin inputs with defaults:

Conversion cost per SF = 6.00

Annual water savings per SF = user-editable default

Calculation basis label = “Water-only payback”

These inputs should instantly recalculate outputs when changed

6. Calculation Logic

For current selected polygons:

Total Area = sum of all selected polygon square footage

Estimated Conversion Cost = total area × conversion cost per SF

Estimated Annual Water Savings = total area × annual water savings per SF

Estimated Payback Period = conversion cost ÷ annual water savings

If annual water savings is zero or blank:

show payback as — or Not available

7. Planning Summary Card

Show a clear summary card with:

Selected polygon count

Total selected area

Conversion cost assumption

Water savings assumption

Estimated conversion cost

Estimated annual water savings

Estimated payback period

This should feel board-friendly, not GIS-heavy
