Slice 1 — Summary Cards → Interactive Filters
What & Why

Convert the top summary cards (Overdue / Active / Requests / Completed) from static counts into primary filtering controls. This removes duplication and gives the user a fast way to change context.

Scope
Add activeSummaryFilter state:
'all' | 'overdue' | 'active' | 'requests' | 'completed'
Make each summary card:
tappable
visually “selected” when active
Sync filter logic:
selecting a summary card updates task list
selecting “All” resets filter
Selected styling:
slightly darker background
stronger border or shadow
keep within current design system
Done Looks Like
Tapping “Requests (4)” shows only requests
Tapping “Overdue” shows overdue only
Selected card is visually obvious
No lag between tap and list update
Acceptance Tests
Tap each card → list updates correctly
Only one card active at a time
Reset to “All” works
No visual glitches between states
Guardrails
Do not change card layout or spacing
Do not introduce new colors
Keep animation subtle (opacity / background only)
